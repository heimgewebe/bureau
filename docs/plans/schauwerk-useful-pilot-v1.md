## Decision

Schauwerk has completed its foundation and controlled Miro proof, but it is not a complete product. The next programme converts the verified registry and first static Grabowski projection into a useful, reviewable and durable operator surface without collapsing source truth into visual output.

## Current verified baseline

- SW-003 controlled live write proof is closed with sanitized evidence.
- SW-004 has complete source, project, surface, view, region, policy and publication contracts.
- The registry is cross-reference checked and has a deterministic digest and inspection CLI.
- SW-005 has a first deterministic Grabowski operator-context projection with committed evidence.
- SW-009 has simulation, live-gate and candidate-check receipts, but no automatic live executor.

## Delivery phases

### Phase A — useful pilot completion

1. Add bounded live operational sources for Grabowski hosts, runtime health, current work and known gaps.
2. Prove the data and view contracts with a second software project.
3. Complete education variants and offline output.
4. Consolidate the visual grammar used by software and education views.

### Phase B — controlled operation

5. Implement a separately approved SW-009 live executor with exact region, revision, snapshot, postflight and restore bindings.
6. Build Regie as the normal review surface for proposal, semantic/visual diff, partial approval, apply, verification and restore.
7. Add overview and live views that remain useful when Miro is unavailable.

### Phase C — presentation and publication

8. Build Bühne outputs for HTML, PDF, PowerPoint, notes, timing and offline packages.
9. Build Schaufenster publication preview, privacy checks, immutable bundles, expiry and withdrawal.

### Phase D — integration and durable operation

10. Add source adapters with explicit authority, freshness, fixture and failure contracts.
11. Add safe maintenance proposals for stale managed regions and open decisions.
12. Add optional cross-project search and semantic suggestions.
13. Add deployment profiles, metrics, backup, restore drills, OAuth rotation and kill-switch tests.

## Safety invariants

- GitHub, repositories and declared source systems remain authoritative.
- Registry declarations and rendered views grant no mutation authority by themselves.
- Human-owned or cooperative regions are never silently replaced.
- Productive writes require an allowlisted target, expected revision, before snapshot, typed operation set, postflight, restore evidence and reviewed receipt.
- Public bundles require explicit source visibility and publication declarations.
- Semantic services remain optional and cannot fabricate fresh facts.

## Exit condition

The initiative is complete when the first useful pilot works from declared live/static sources through Regie, presentation and publication surfaces, while source failure, provider outage and faulty mutation have tested recovery paths. Individual later-stage tasks may be split into narrower successor initiatives when their own implementation plans become authoritative.
